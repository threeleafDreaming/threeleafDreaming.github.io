【工具】Git-码农“吃饭的碗”要拿好

汝之观览，吾之幸也！本文主要讲解的是Git的轻巧使用（创建、下载、上传、更新、回退），我们平常都是通过idea自带的git工具，或者其他工具来拉取提交代码，这里主要用命令行的方式拉取代码，并配合不同场景进行整理命令执行步骤。

一、概念

是一个开源的分布式版本控制系统，可以有效、高速地处理从很小到非常大的项目版本管理。

二、安装部署

三、常用命令

1、git config

配置用户名与邮箱

2、git version

查看git的版本信息

3、git init

初始化git本地仓库

4、git clone

克隆远程分支

5、git add

将修改过的文件和新文件添加到暂存区

6、git commit

提交修改的文件到本地仓库

7、git status

在git add和git commit之间查看状态

8、git branch

查看git全部分支

删除git分支

9、git checkout

切换到其他分支

从当前分支拆出一个本地分支

10、git remote

将本地分支与远程分支关联上

11、git pull

拉取远程分支代码到本地

12、git push

推送本地代码到远程

13、git fetch

查看全部的远程分支

14、git log

获取git提交的日志

15、git merge

合并分支代码到本地

16、git tag

将分支进行打标签，打标签后的分支不可更改代码，常用于部署生产使用

17、git diff

比较两个分支

18、git help

帮助命令，会列出全部命令的使用方式

参考文章：最常用的 35 个 Git 命令
